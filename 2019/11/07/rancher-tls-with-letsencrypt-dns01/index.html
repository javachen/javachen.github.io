Rancher使用letsEncrypt生成证书并做DNS01校验

2019-11-07
kubernetes
约 1696 字 预计阅读 4 分钟 次阅读

前面使用Helm安装Rancher HA集群，配置证书的时候，因为没有权威证书，所以测试过利用Rancher和自签名文件来生成证书，但是生成的证书在浏览器不被信任。当时，也测试过通过letsEncrypt来生成，但是没有配置成功。把Rancher的Chart源码阅读之后，将源码修改了，增加了DNS01校验方式，最后测试成功。

在阅读本文章之前，先阅读以下两篇文章：

  • 使用Helm安装Rancher HA集群

  • 使用Cert Manager配置Let’s Encrypt证书

{{- if eq .Values.tls "ingress" -}}

apiVersion: certmanager.k8s.io/v1alpha1
kind: Issuer
metadata:
  name: {{ template "rancher.fullname" . }}
  labels:
    app: {{ template "rancher.fullname" . }}
    chart: {{ .Chart.Name }}-{{ .Chart.Version }}
    heritage: {{ .Release.Service }}
    release: {{ .Release.Name }}
spec:
  acme:
    {{- if eq .Values.letsEncrypt.environment "production" }}
    server: https://acme-v02.api.letsencrypt.org/directory
    {{- end }}
    {{- if eq .Values.letsEncrypt.environment "staging" }}
    server: https://acme-staging-v02.api.letsencrypt.org/directory
    {{- end }}
    email: {{ .Values.letsEncrypt.email }}
    privateKeySecretRef:
      name: letsencrypt-{{ .Values.letsEncrypt.environment }}
    http01: {}
  {{- end -}}
{{- end -}}

可以看到letsEncrypt使用的是http01校验，因为服务器上没有配置web服务器，所以用该方式校验比较麻烦，故乡改为dns01方式校验。

添加DNS01校验

参考 使用Cert Manager配置Let’s Encrypt证书 这篇完整，我们可以利用 cert-manager-webhook-dnspod 通过webhook的方式进行dns01校验。

其创建issuer代码如下：

apiVersion: certmanager.k8s.io/v1alpha1
kind: ClusterIssuer
metadata:
  name: letsencrypt-staging
spec:
  acme:
    server: https://acme-staging-v02.api.letsencrypt.org/directory
    email: junecloud@163.com
    privateKeySecretRef:
      name: letsencrypt-staging
    solvers:
    - selector:
        dnsNames:
        - '*.javachen.space'
      dns01:
        webhook:
          groupName: acme.javachen.space 
          solverName: dnspod
          config:
            apiID: 123438  #DNSPOD 的 apiID，注意替换
            apiTokenSecretRef:
              key: api-token
              name: dnspod-credentials

    {{- if eq .Values.letsEncrypt.environment "prod" }}
    server: https://acme-v02.api.letsencrypt.org/directory
    {{- end }}
    {{- if eq .Values.letsEncrypt.environment "staging" }}
    server: https://acme-staging-v02.api.letsencrypt.org/directory
    {{- end }}
    email: {{ .Values.letsEncrypt.email }}
    privateKeySecretRef:
      name: {{ template "rancher.fullname" . }}-letsencrypt-{{ .Values.letsEncrypt.environment }}
{{- if .Values.letsEncrypt.solvers  }}   
    solvers:  
{{ toYaml .Values.letsEncrypt.solvers | indent 4 }}
{{- else }}
    http01: {}
{{- end }}    
  {{- end -}}
{{- end -}}

注意：

1、这里连带把production改为了prod

2、将Issuer修改为ClusterIssuer

3、将metadata的name修改为{{ template "rancher.fullname" . }}-letsencrypt-{{ .Values.letsEncrypt.environment }}

4、privateKeySecretRef的name修改为{{ template "rancher.fullname" . }}-letsencrypt-{{ .Values.letsEncrypt.environment }}

因为将Issuer修改为ClusterIssuer，所以也需要修改Ingress中注释：

  • 将certmanager.k8s.io/issuer改为certmanager.k8s.io/cluster-issuer
  • 将certmanager.k8s.io/cluster-issuer的值改为{{ template "rancher.fullname" . }}-letsencrypt-{{ .Values.letsEncrypt.environment }}

对应rancher/templates/ingress.yaml代码：

1
2
3

  {{- if ne .Values.ingress.tls.source "secret" }}
    certmanager.k8s.io/cluster-issuer: {{ template "rancher.fullname" . }}-letsencrypt-{{ .Values.letsEncrypt.environment }}
  {{- end }}

修改rancher/values.yaml，在environment下面添加solvers: []，这里连带把production改为了prod

3

  environment: prod
  solvers: []

cat <<EOF > rancher-values.yaml
hostname: rancher.javachen.space
ingress:
  tls:
    source: letsEncrypt
letsEncrypt:
  email: junecloud@163.com
  environment: prod
  solvers: 
  - selector:
      dnsNames:
      - 'rancher.javachen.space'
    dns01:
      webhook:
        groupName: acme.javachen.space
        solverName: dnspod
        config:
          apiID: 123438  #DNSPOD 的 apiID，注意替换
          apiTokenSecretRef:
            key: api-token
            name: dnspod-credentials 
EOF

从本地chart文件安装：

1
2

helm install --name rancher --namespace cattle-system \
	./rancher --version v2.3.2 -f rancher-values.yaml
