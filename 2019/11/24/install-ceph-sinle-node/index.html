本文主要记录在虚拟机中安装Ceph单节点集群的过程，其实多配置几个节点，也就是安装集群的过程。

集群规划

集群环境：

主机 IP 操作系统 软件 组件
k8s-rke-node001 192.168.56.111 Centos7.7.1908 Ceph 14.2.4 ceph_deploy、OSD.0、MON

安装用户：chenzj

配置环境

设置hosts

1

192.168.56.111 k8s-rke-node001

创建安装用户

在每个节点创建普通用户如chenzj，并设置密码和sudo权限

1
2
3
4

USER=chenzj
useradd -G docker -d /home/$USER -m $USER 
echo $USER|passwd $USER --stdin >/dev/null 2>&1
sudo echo "$USER ALL = (root) NOPASSWD:ALL" | sudo tee /etc/sudoers.d/$USER

设置TTY

执行命令visudo修改suoders文件：

  1. 注释Defaults requiretty

Defaults requiretty修改为 #Defaults requiretty， 表示不需要控制终端。

否则会出现sudo: sorry, you must have a tty to run sudo

  1. 增加行 Defaults visiblepw

否则会出现 sudo: no tty present and no askpass program specified

1
2

sed -i 's/Defaults *requiretty/#Defaults requiretty/g' /etc/sudoers
sed -i 's/Defaults *!visiblepw/Defaults   visiblepw/g' /etc/sudoers

配置无密码登陆

设置chenzj用户无密码登陆：

1
2
3
4

su - chenzj
sudo yes|ssh-keygen -f ~/.ssh/id_rsa -t rsa -N ""

ssh-copy-id k8s-rke-node001

设置ssh默认用户名：

1
2
3
4
5
6
7

cat << EOF >> ~/.ssh/config
Host      k8s-rke-node001
Hostname  k8s-rke-node001
User         chenzj
EOF

chmod 644 ~/.ssh/config

sudo yum install yum-plugin-priorities -y

加载rbd模块

sudo modprobe rbd

配置ceph-deploy节点

在管理节点 k8s-rke-node001 上配置yum源，注意：这里ceph使用的是nautilus版本。

安装部署工具：

1
2
3
4

# 安装 EPEL 源
sudo yum install -y epel-release

sudo yum install ceph-deploy -y

说明：ceph-deploy 工具需要使用 ssh 来自动化部署 Ceph 各个组件，因此需要保证部署节点能够免密码登录待部署节点；最后，待部署节点最好加入到部署节点的 hosts 中，方便使用域名(某些地方强制)连接管理。

创建集群配置

ceph-deploy 工具部署集群前需要创建一些集群配置信息，其保存在 ceph.conf 文件中，这个文件未来将会被复制到每个节点的 /etc/ceph/ceph.conf

在ceph-deploy节点执行：

1
2
3
4
5
6
7

# 创建集群配置目录
mkdir ceph-cluster && cd ceph-cluster

# 创建 monitor-node
ceph-deploy new k8s-rke-node001 \
  --public-network 192.168.56.0/24 \
  --cluster-network 192.168.56.0/24

注意：public-network要和当前ip的网段保持对应。

出现异常：

安装相关依赖即可：

1

sudo yum install python-pkg-resources python-setuptools -y

执行完之后，在当前目录下会创建一些辅助文件：

接下来，我们需要修改下 ceph.conf 配置文件，增加副本数为 1，因为我们目前只有1个 osd 节点。

 1
 2
 3
 4
 5
 6
 7
 8

10

echo "
osd_pool_default_size = 1

[mon]
mon_allow_pool_delete = true
mon_max_pg_per_osd = 1024
#时钟同步
mon_clock_drift_allowed = 2
mon_clock_drift_warn_backoff = 30
" >> ceph.conf

如果是在集群启动后修改配置文件，则需要同步配置文件：

1

ceph-deploy --overwrite-conf config push k8s-rke-node001

创建集群

创建集群使用 ceph-deploy 工具即可，通过 ceph-deploy 在各个节点安装 ceph：

1
2

# 安装 ceph
ceph-deploy install k8s-rke-node001

此过程需要等待一段时间，因为 ceph-deploy 会 SSH 登录到各 node 上去，依次执行安装 ceph 依赖的组件包。

如果下载太慢，可以设置镜像：

1
2

sudo rpm --import http://mirrors.163.com/ceph/keys/release.asc
ceph-deploy install --repo-url http://mirrors.163.com/ceph/rpm-nautilus k8s-rke-node001

如果还是太慢，则在每个节点配置ceph的yum源，然后通过yum安装：

1

sudo yum -y install ceph ceph-radosgw

再次执行：

1

ceph-deploy install k8s-rke-node001

安装成功之后，检查每个节点的ceph版本是否一致：

1
2

sudo ceph --version
ceph version 14.2.4 (75f4de193b3ea58512f204623e6c5a16e6c1e1ba) nautilus (stable)

初始化 monitor node 和 秘钥文件：

1

ceph-deploy mon create-initial

注意：执行完成后会在/etc/ceph目录多以下内容：

如果出现下面异常：

1
2

[ceph_deploy.mon][ERROR ] Some monitors have still not reached quorum:
[ceph_deploy.mon][ERROR ] k8s-rke-node001

可能原因：

  • ceph版本不一致
  • hostname没设置正确

查看仲裁信息：

1

ceph quorum_status --format json-pretty

部署ceph cli工具和秘钥

我们先通过ceph admin命令将将ceph.client.admin.keyring同步到各个Node上，以便可以在各个Node上使用ceph命令连接到monitor：

1

ceph-deploy admin k8s-rke-node001

各节点修改ceph.client.admin.keyring权限：

1

sudo chmod +r /etc/ceph/ceph.client.admin.keyring

清理环境

如果我们在安装过程中出现了问题，需要重新操作的时候，例如想清理我搭建的这个集群的话，可以使用以下命令。

1
2
3
4
5
6
7
8

# ceph-deploy 上执行

# 清理配置
ceph-deploy purgedata k8s-rke-node001
ceph-deploy forgetkeys

# 清理 Ceph 安装包
ceph-deploy purge k8s-rke-node001

安装mgr dashboard

自 nautilus开始，dashboard作为一个单独的模块独立出来了，使用时需要单独安装:

1

sudo yum install -y ceph-mgr-dashboard

启用dashboard：

1

ceph mgr module enable dashboard --force

默认启用SSL/TLS，所以需要创建自签名根证书：

1

ceph dashboard create-self-signed-cert

创建具有管理员角色的用户

1

ceph dashboard ac-user-create admin admin administrator

扩容集群

1

ceph-deploy mon add k8s-rke-node002

查看仲裁信息：

1

ceph quorum_status --format json-pretty
  • 请确保ntp时钟同步

添加MGR

1

ceph-deploy mgr create k8s-rke-node002

查看standby manager：

1

ceph -s

配置客户端

1
2
3

ssh-copy-id k8s-rke-node002

ceph-deploy install k8s-rke-node002

将Ceph 配置文件复制到客户端：

1

ceph-deploy config push k8s-rke-node002

新建用户密钥

客户机需要 ceph 秘钥去访问 ceph 集群。ceph创建了一个默认的用户client.admin，它有足够的权限去访问ceph集群。但是不建议把 client.admin 共享到所有其他的客户端节点。这里我用分开的秘钥新建一个用户k8s去访问特定的存储池。

1

ceph auth get-or-create client.k8s mon 'allow r' osd 'allow class-read object_prefix rbd_children, allow rwx pool=rbd'

为 k8s-rke-node002 上的 client.rbd 用户添加秘钥：

1

ceph auth get-or-create client.k8s | ssh k8s-node4 tee /etc/ceph/ceph.client.k8s.keyring

到客户端 k8s-rke-node002 检查集群健康状态：

1
2

cat /etc/ceph/ceph.client.k8s.keyring >> /etc/ceph/keyring
ceph -s --name client.k8s
