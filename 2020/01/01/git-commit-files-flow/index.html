创建一个新项目

Git 全局设置：

1
2

git config --global user.name "chenzj"
git config --global user.email "junecloud@163.com"

创建一个仓库

下载Github上创建好的空项目，添加README.md：

1
2
3
4
5
6

git clone git@github.com:javachen/test_git.git
cd test_git
touch README.md
git add README.md
git commit -m "add README"
git push -u origin master

推送现有文件夹

1
2
3
4
5
6
7

cd existing_folder
echo "# test_git" >> README.md
git init
git add README.md
git commit -m "first commit"
git remote add origin git@github.com:javachen/test_git.git
git push -u origin master

推送现有的 Git 存储库

1
2
3
4
5

cd existing_repo
git remote rename origin old-origin
git remote add origin git@github.com/javachen/test_git.git
git push -u origin --all
git push -u origin --tags

git diff
git diff <file>     			# 比较当前文件和暂存区文件差异
git diff <$id1> <$id2>   	# 比较两次提交之间的差异 
git diff <branch1> <branch2>   # 在两个分支之间比较  
git diff --staged   			# 比较暂存区和版本库差异 
git diff --cached   			# 比较暂存区和版本库差异 
git diff --stat     			# 仅仅比较统计信息 
git diff HEAD 						# 输出工作区、暂存区和本地最近的版本的不同 
git diff "@{yesterday}"   # 查看昨天的改变 
git diff 1b6d "master~2"  # 查看一个特定版本与倒数第二个变更之间的改变

git log							# 查看所有提交日志
git log <file>      # 查看该文件每次提交记录
git log -p <file>   # 显示版本历史，以及版本间的内容差异

git log -1 HEAD~3          # 显示比HEAD早3个提交的那个提交
git log -1 HEAD^^^				 # 显示比HEAD早3个提交的那个提交

git log Branch1 ^Branch2 	 # commit历史中显示Branch1有的，但是Branch2没有commit
git log --pretty=oneline --graph --decorate --all	# 展示简化的 commit 历史
git log --all --grep='add'	# 通过 grep 查找

git grep 'admin'	 					# 查询字符串
git reflog				          # 查看操作记录
git blame <file-name> 			# 查看某段代码是谁写的
git status --ignored 			  # 展示忽略的文件#
git shortlog -sn 					  # 统计每个人提交的次数

git whatchanged --since="2 weeks ago"  # 查看两周内修改的文件

如果想放弃上次的提交，则：

1
2

$ git reset --hard  HEAD^
HEAD 现在位于 bbb83f6 add 2

如果想回退到某一次提交，如 b11f0afe2d400c55dcf4481da803879df192aea9 这个commit，则：

1

git reset --hard b11f0afe2d

如果想回退远程仓库的状态，则：

1

git reset --hard origin/master

git reset --hard origin/master 		# 回到远程仓库的状态
git reset --hard  HEAD^ 					# 恢复最近一次提交过的状态，即放弃上次提交后的所有本次修改
git reset --hard <commit id>  		# 恢复到某一次提交的状态
git reset HEAD <file> 						# 抛弃暂存区中文件的修改
git reset <file>    							# 从暂存区恢复到工作文件
git reset -- .      							# 从暂存区恢复到工作文件

# 把所有的改动都重新放回工作区，并清空所有的 commit，这样就可以重新提交第一个 commit 了
git update-ref -d HEAD

再次来修改README.md，当对README.md的修改还没有完成的时候，发现 hello.md 中有一个bug需要修复，这时候可以将当前所有的修改暂存起来，然后去修改 hello.md 并提交。

$ git add .

$ git ci
[master eeb2375] fix hello.md
 1 file changed, 1 insertion(+), 1 deletion(-)
 
# 恢复暂存的内容 
$ git stash apply
