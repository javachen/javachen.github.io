压缩包安装

下载文件

从 Keycloak downloads 下载安装包，安装包有两种格式：zip、tar.gz，然后解压：

1
2

wget https://downloads.jboss.org/keycloak/9.0.0/keycloak-9.0.0.zip
unzip keycloak-9.0.0.zip

启动服务

进入到解压后目录的bin目录下，启动服务：

1
2

cd keycloak-9.0.0/bin
./standalone.sh -Djava.net.preferIPv4Stack=true

默认启动端口是8080，可以添加参数修改启动端口，如下面参数是在8080端口上添加100，即为8180.

1

./standalone.sh -Djboss.socket.binding.port-offset=100

另外，Keycloak有一些实验中的特性，参考：https://www.keycloak.org/docs/latest/server_installation/#profiles：

Name Description Enabled by default Support level
account2 New Account Management Console No Experimental
account_api Account Management REST API No Preview
admin_fine_grained_authz Fine-Grained Admin Permissions No Preview
docker Docker Registry protocol No Supported
impersonation Ability for admins to impersonate users Yes Supported
openshift_integration Extension to enable securing OpenShift No Preview
scripts Write custom authenticators using JavaScript No Preview
token_exchange Token Exchange Service No Preview
upload_scripts Upload scripts through the Keycloak REST API No Deprecated

如果想开启所有特性，添加下面启动参数：

1

-Dkeycloak.profile=preview

开启某一个特性，可以使用下面参数：

1

-Dkeycloak.profile.feature.<feature name>=enabled

例如，开启scripts特性：

1

./standalone.sh -Djboss.socket.binding.port-offset=100 -Dkeycloak.profile.feature.scripts=enabled

创建管理员用户

创建管理员用户：

1

./add-user-keycloak.sh -u admin -p admin123

访问 http://localhost:8180/auth/admin/ ，登陆Admin Console，输入用户名和密码。

Docker安装

安装Postgres数据库：

1
2
3

docker run -d --name postgres -p 5432:5432 -e POSTGRES_PASSWORD=123456 postgres

docker exec -it postgres psql -Upostgres -w -c "CREATE DATABASE keycloak WITH OWNER postgres;"

使用docker安装：

docker run -d \
  --name keycloak \
  -e KEYCLOAK_USER=admin \
  -e KEYCLOAK_PASSWORD=admin123 \
  -e DB_VENDOR=POSTGRES \
  -e DB_ADDR=192.168.1.111 \
  -e DB_PORT=5432 \
  -e DB_DATABASE=keycloak \
  -e DB_USER=postgres \
  -e DB_PASSWORD=123456 \
  -e JDBC_PARAMS='connectTimeout=30' \
  -p 8080:8080 \
  jboss/keycloak

在此之后，我们只需登录到容器并导航到bin文件夹。

1
2

docker exec -it keycloak /bin/bash
cd keycloak/bin

我们需要从CLI客户端登录keycloak服务器，之后我们不再需要身份验证：

1

./kcadm.sh config credentials --server http://localhost:8080/auth --realm master --user admin --password admin123

version: '3'

  postgres:
      image: postgres
      volumes:
        - ./data:/var/lib/postgresql/data
      environment:
        POSTGRES_DB: keycloak
        POSTGRES_USER: postgres
        POSTGRES_PASSWORD: 123456
  keycloak:
      image: quay.io/keycloak/keycloak:latest
      environment:
        DB_VENDOR: POSTGRES
        DB_ADDR: postgres
        DB_DATABASE: keycloak
        DB_USER: postgres
        DB_SCHEMA: public
        DB_PASSWORD: 123456
        KEYCLOAK_USER: admin
        KEYCLOAK_PASSWORD: admin123
        #JDBC_PARAMS: "ssl=true"
      ports:
        - 8080:8080
      depends_on:
        - postgres

Helm安装

创建命名空间：

1

kubectl create namespace keycloak

创建TLS证书：

 1
 2
 3
 4
 5
 6
 7
 8

15
16

cat << EOF | kubectl create -f -   
apiVersion: cert-manager.io/v1alpha2
kind: Certificate
metadata:
  name: keycloak-test-javachen-com-cn-cert
  namespace: keycloak
spec:
  secretName: keycloak-test-javachen-com-cn-cert
  renewBefore: 720h
  dnsNames:
  - "keycloak.javachen.com"
  - "keycloak.test.javachen.com"
  issuerRef:
    name: cert-manager-webhook-dnspod-cluster-issuer
    kind: ClusterIssuer
EOF

cat <<EOF > keycloak-values.yaml
keycloak:
  image: 
    tag: latest
  username: admin
  password: admin123
  extraArgs: "-Dkeycloak.profile.feature.upload_scripts=enabled"
  persistence:
    deployPostgres: false
    dbVendor: postgres
    dbName: keycloak
    dbHost: 192.168.1.111
    dbPort: 5432 
    dbUser: postgres
    dbPassword: "123456"
  ingress:
    enabled: true
    annotations:
      kubernetes.io/ingress.class: nginx
      nginx.ingress.kubernetes.io/ssl-redirect: "true"
      nginx.ingress.kubernetes.io/proxy-body-size: 100m
    hosts:
      - keycloak.javachen.com
      - keycloak.test.javachen.com
    tls:
      - secretName: keycloak-test-javachen-com-cn-cert
        hosts:
          - keycloak.javachen.com
          - keycloak.test.javachen.com
EOF

开启一些实验特效，参考：https://www.keycloak.org/docs/latest/server_installation/#profiles

1

extraArgs: "-Dkeycloak.profile.feature.upload_scripts=enabled"

helm3安装：

1

helm install keycloak -n keycloak -f keycloak-values.yaml ./keycloak

卸载：

1

helm del keycloak -n keycloak
